# TASK: Contributing Guide

## Goal
Create CONTRIBUTING.md with developer setup instructions, code conventions, testing guidelines, and PR process.

## Scope

### Allowed
- CONTRIBUTING.md

### Forbidden
- Any source code changes

## Requirements
1. Prerequisites: Node 20+, pnpm, git
2. Dev setup: clone, install, build, test (copy-paste commands)
3. Project structure overview (packages and their roles)
4. File organization conventions (reference CLAUDE.md rules)
5. Code style: TypeScript strict, ESM, JSDoc, named exports
6. Testing: how to run tests, where to put them, ink-testing-library for UI
7. Commit conventions: conventional commits format with examples
8. Branch workflow: feature branches from main, PR required
9. PR template: what to include (description, test plan, screenshots if UI)
10. Code review expectations

## Definition of Done
- [ ] CONTRIBUTING.md is comprehensive and accurate
- [ ] Dev setup instructions work from scratch on a clean clone
- [ ] All conventions match what CLAUDE.md specifies
- [ ] Language: English
